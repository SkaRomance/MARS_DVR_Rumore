## Acceptance criteria globali MVP

Il progetto ГЁ MVP-complete quando TUTTI i seguenti sono verificati:

1. **Flusso consulente end-to-end**
   - Consulente apre MARS web вҶ’ seleziona DVR Azienda X вҶ’ click "Modulo Rumore" вҶ’ ModuleFrame iframe apre Rumore UI con JWT
   - Rumore importa automaticamente workPhases + phaseEquipments dal DVR
   - AI Autopilot completa valutazione in < 90s per DVR medio (5 fasi, 12 equipment)
   - Consulente revisa, approva/rigetta ogni suggestion individualmente o in bulk
   - Consulente salva вҶ’ PUT su MARS `module_extensions.noise` aggiornato

2. **QualitГ  AI**
   - Confidence media autopilot > 0.75 su golden dataset
   - Review agent flaggia almeno 80% dei casi anomali nel golden
   - Narrativa generata passa check compliance italiana (no citazioni errate)

3. **Compliance**
   - Ogni campo ha `origin` enum tracciabile
   - Disclaimer AI obbligatorio in export DOCX
   - AuditTrailPanel mostra diff AI vs user con export CSV

4. **Robustezza**
   - Healthcheck 5/5 green (DB, Redis, Ollama, MARS, ChromaDB)
   - Scheduler esegue 1 ciclo completo senza errori
   - Load test: 50 autopilot concurrent session senza OOM/crash

5. **Deploy readiness**
   - Docker image non-root, resource limits, <500MB
   - CI passa lint+test+build+scan
   - Docs operations: secrets rotation + rollback + backup restore

## Rollback strategy

Ogni wave ha piano di rollback esplicito nel suo plan file. Pattern generale:

| Wave | Rollback | Impact |
|---|---|---|
| W24 | `git revert` su branch MARS `noise-module-integration`; modulo funziona con MARS pre-esistente solo se usa solo endpoint legacy | Rumore funziona in standalone mode |
| W25 | `alembic downgrade` ai revision number precedente | DB tornato al pre-refactor |

| W27 | Disabilita `AI_AUTOPILOT_ENABLED=false` in env | UI mostra solo modalitГ  manuale |
| W28 | `docker-compose stop scheduler` | No scraping auto, data si fa stale |
